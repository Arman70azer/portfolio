chess

A mobile app that allows users to play chess with others in real-time. Rust:

smart-road

Smart traffic control system for autonomous vehicles (AVs) without using traditional traffic lights. The system ensures safe passage through intersections by managing vehicle speeds, distances, and directions. Features: Vehicle Physics: Implemented velocity, distance, and safe distance rules for AVs, Smart Intersection: Vehicles follow lanes and routes, adjusting speeds based on real-time traffic and avoiding collisions, Animation: Created realistic vehicle movement and turning animations, Vehicle Creation: Vehicles are generated using keyboard commands, with randomized routes and velocities, Statistics: The simulation tracks key metrics like vehicle throughput, max/min velocity, and close calls to evaluate performance, This project demonstrates my ability to simulate real-world scenarios and develop intelligent algorithms for managing traffic and ensuring safety in autonomous driving environments.

multiplayer-fps

Maze Wars with a client-server architecture, using UDP for communication. The game features a mini-map, player positions, and a frame rate display, all while maintaining the original game's graphical elements like walls and players. Key Features: - User Interface: Includes a mini-map, game graphics, and frame rate display. - Client-Server: Supports multiple clients (up to 10) connecting to a central server, with IP-based connection and username input. - Levels: Three difficulty levels with increasingly complex mazes. - Performance: Maintains a frame rate of over 50 fps. The project also explores game mechanics, GUI design, and network protocols, providing a seamless multiplayer experience.

0-shell

Simple Unix shell, similar to BusyBox, which supports essential commands such as `echo`, `cd`, `pwd`, `ls`, `cat`, `cp`, `rm`, `mv`, `mkdir`, and `exit`. The shell executes commands by creating new processes and manages errors with custom error messages. Key Features: - Command Execution: Supports basic commands, process creation, and synchronization. - Error Handling: Displays relevant error messages when commands fail. - Process Control: Handles program interruption via `Ctrl+D`. - File System Operations: Manages simple file operations using system calls. This project enhanced my understanding of Unix systems, process management, and command-line interfaces.
